NL2Cypher 風險與防護機制（Guardrail）

主要風險

  • 幻覺欄位：模型生成不存在的 label/property
  • 查詢注入：夾帶 CREATE / DELETE / CALL 等危險語句
  • 歧義問句：缺少主體時誤查錯誤疾病

失敗處理

若 guardrail 不通過，回覆「需要更明確條件」，並附可接受問題範例，不直接執行查詢。

防護策略

  • 白名單意圖：只允許固定 intent 集合
  • 模板查詢：禁用自由生成 Cypher
  • 唯讀限制：僅 MATCH/RETURN，拒絕寫入語句
  • Schema 驗證：未知 label / relation 立即阻擋
  • trace 記錄：保留 intent、cypher、攔截原因
