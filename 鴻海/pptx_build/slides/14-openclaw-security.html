OpenClaw 技能生態風險與安全審查機制

已知風險面

  • Plugin/skill 皆可能取得主機執行權限
  • npm install 時的 lifecycle script 供應鏈風險
  • 開放 DM/群組 + 工具啟用時的 prompt injection
  • 技能檔案被篡改（watcher 熱更新）

審查策略核心

以「最小權限 + 可驗證 + 可追溯」為原則，避免惡意 skill 在未審核下執行。

技能安全審查流程（6階段）

  • 1. 提交：含 manifest、schema、來源資訊
  • 2. 靜態掃描：危險 API / shell / network 規則
  • 3. 行為沙箱：受限檔案與工具白名單
  • 4. 人工簽核：安全 reviewer + 風險分級
  • 5. 簽章發布：版本鎖定 + 哈希校驗
  • 6. 監控回滾：異常行為告警與快速停用
