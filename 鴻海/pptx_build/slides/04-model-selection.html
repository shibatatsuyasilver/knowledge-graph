開源模型選型：為何使用 Qwen2.5-7B-Instruct

模型

優點

限制

Qwen2.5-7B-Instruct

中英表現平衡、工具/JSON 指令遵循佳、社群部署資源完整。

7B 仍有推理天花板，長文本與複雜邏輯需 guardrail。

Llama 3.1 8B Instruct

生態成熟、通用任務穩定。

中文與格式化輸出穩定性略受 prompt 影響。

Mistral 7B Instruct

速度快、部署資源小。

領域抽取與中文術語一致性需額外後處理。

選型結論

  • 本題需求強調「抽取 + 查詢轉換 + 中文解說」
  • Qwen2.5-7B 在單機成本與輸出可控性之間平衡最佳
  • 搭配低溫度與 JSON schema 可顯著降低格式漂移

預期部署規格（原型）

  • Ollama + qwen2.5:7b-instruct（4-bit）
  • CPU 可跑，GPU 更佳（降低延遲）
  • 每次查詢目標 1~3 秒內回覆
